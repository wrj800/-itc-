## 二、当前项目定位

项目名称：企业 ICT 智能运维与故障诊断平台。

项目目标：

1. 管理企业 ICT 设备与运维对象。
2. 通过 Prometheus、Exporter、Blackbox Exporter 采集运行状态。
3. 支持真实 Linux 虚拟节点监控，不停留在纯前端页面模拟。
4. 后续接入 AI 服务，实现告警摘要、故障根因分析、处置建议和知识库问答。

## 三、当前技术栈

| 层级 | 技术 |
|---|---|
| 前端 | Vue 3、Vite、TypeScript、Element Plus、ECharts |
| 后端 | Spring Boot 3、JDK 17、Maven |
| AI 服务 | Python、FastAPI |
| 数据库与缓存 | MySQL、Redis |
| 监控采集 | Prometheus、node_exporter、Blackbox Exporter |
| 真实节点 | WSL2 Ubuntu 24.04 + node_exporter |
| 部署辅助 | Docker Desktop、Docker Compose |

## 五、已实现接口

### 5.1 健康检查

```http
GET /api/health
```

作用：检查 Spring Boot 后端是否可用。

### 5.2 Prometheus 查询

```http
GET /api/monitor/prometheus/query?source=docker&query=up
GET /api/monitor/prometheus/query?source=wsl&query=up
```

参数说明：

| 参数 | 说明 |
|---|---|
| `source` | 数据源，当前支持 `docker` 和 `wsl` |
| `query` | PromQL 查询语句 |

### 5.3 Prometheus 目标查询

```http
GET /api/monitor/prometheus/targets?source=docker
GET /api/monitor/prometheus/targets?source=wsl
```

作用：查询 Prometheus 当前采集目标。

## 六、当前监控数据源

| 数据源 | 地址 | 用途 |
|---|---|---|
| `docker` | `http://127.0.0.1:9090` | Docker 观测栈 |
| `wsl` | `http://<WSL-IP>:19090` | 真实 Linux 虚拟节点 |

WSL IP 可能变化，`scripts/start-backend.ps1` 会自动设置：

```text
ICT_PROMETHEUS_WSL_BASE_URL=http://<WSL-IP>:19090
```
