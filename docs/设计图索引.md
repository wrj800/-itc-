## 四、当前设计顺序

1. 先完成页面线框图，确定系统有哪些页面。
2. 再完成业务流程图，确定数据和操作如何流转。
3. 再完成 UML 图，明确角色、对象、时序、部署和数据关系。
4. 最后再依据设计图继续实现前端页面和后端接口。

## 五、后续实现依据

后续开发时，我会以这些设计图为依据推进：

1. 设备资产模块优先对应 `ict_device`、`monitor_target`。
2. 监控目标模块优先对应 Prometheus 查询和采集目标管理。
3. 告警中心优先对应 `alarm_record`。
4. 工单管理优先对应 `work_order`、`work_order_log`。
5. AI 诊断优先对应 FastAPI 服务、知识库和大模型 API。
