本文档用于记录毕业设计开发过程中的阶段推进、技术选择、问题定位、解决方案和验证证据。与 `开发记录.md` 不同，本文档更偏向过程性说明，便于后续整理论文、答辩材料和中期检查材料。

## 2026-05-12 阶段一：项目方向确定与可行性验证

### 1. 背景与目标

最初我将项目方向确定为“企业 ICT 智能运维与故障诊断平台”。我的毕业设计目标不是只完成静态页面或简单增删改查，而是尽量体现真实运维场景，包括服务器、网络、监控、告警、AI 诊断等内容。

因此本阶段的核心目标不是马上堆业务页面，而是先验证本机是否具备完成该项目的基础条件：

1. 是否可以运行 Java 后端。
2. 是否可以运行 Vue 前端。
3. 是否可以运行 Python/FastAPI AI 服务。
4. 是否可以运行 Docker 观测组件。
5. 是否可以接入真实或近真实的运维采集对象。

### 2. 技术路线确认

最终采用的技术路线如下：

| 模块 | 技术选择 | 选择原因 |
|---|---|---|
| 前端 | Vue 3 + Vite + Element Plus | 开发效率高，适合后台管理系统和监控面板 |
| 后端 | Spring Boot 3 + JDK 17 + Maven | 企业项目常见技术栈，适合毕业设计后端主体 |
| AI 服务 | Python + FastAPI | 便于后续接入大模型 API、RAG 和故障诊断逻辑 |
| 缓存 | Redis | 可用于会话、告警状态、热点数据缓存 |
| 监控 | Prometheus + Exporter | 运维监控领域成熟方案，真实可验证 |
| 真实节点 | WSL2 Ubuntu + node_exporter | 不依赖额外实体服务器，也能采集真实 Linux 指标 |
| 容器环境 | Docker Desktop + Docker Compose | 便于统一启动 Redis、Prometheus、Blackbox 等组件 |

### 3. 可行性判断

经过检查，本机满足当前项目开发条件：

1. BIOS 虚拟化已开启。
2. Docker Desktop 可用。
3. WSL2 Ubuntu 24.04 可用。
4. JDK17 和 Maven 可用。
5. Node.js/npm 可运行 Vue 项目。
6. Python 虚拟环境可安装 FastAPI 依赖。

结论：可以按照“真实 ICT 智能运维平台”方向继续作为毕业设计项目推进。

Docker Prometheus 尝试直接采集 WSL 节点时，出现容器到 WSL NAT 网络不稳定的问题。由于当前环境不方便使用管理员权限配置 `netsh portproxy`，因此没有强行修改系统网络，而是选择在 WSL 内单独部署 Prometheus。

这个方案的优点：

1. WSL 内 Prometheus 可以稳定采集本机 Linux 指标。
2. Spring Boot 后端可以通过 WSL IP 查询 WSL Prometheus。
3. 演示时能证明数据来自真实 Linux 系统，而不是页面模拟。

### 4. 问题定位与修复

启动 WSL Prometheus 时曾出现“脚本提示 already running，但实际没有 Prometheus 进程”的情况。

原因：

原脚本使用的 `pgrep -f prometheus-wsl` 匹配范围过宽，会把当前执行的启动命令本身误判为 Prometheus 进程。

修复方式：

将判断逻辑改为：

1. 使用 `pgrep -x prometheus` 获取真实 Prometheus 进程。
2. 再读取 `/proc/<pid>/cmdline`。
3. 判断命令行中是否包含 `/etc/prometheus-wsl/prometheus.yml`。

修复后验证：

```text
Prometheus Server is Ready.
up{job="wsl-ubuntu-node"} = 1
node_uname_info{nodename="heyan", sysname="Linux"} = 1
```

### 2. 后端问题修复

联调时 Prometheus 查询接口返回 `500`。

日志显示：

```text
Name for argument of type [java.lang.String] not specified,
and parameter name information not available via reflection.
```

原因：

Spring Boot 在当前编译参数下不能通过反射拿到 `@RequestParam` 的方法参数名。

修复方式：

显式指定参数名：

```java
@RequestParam(name = "source", defaultValue = "docker")
@RequestParam(name = "query", defaultValue = "up")
```

修复后验证：

```text
GET /api/monitor/prometheus/query?source=docker&query=up 成功
GET /api/monitor/prometheus/query?source=wsl&query=up 成功
GET /api/monitor/prometheus/query?source=wsl&query=node_uname_info 成功
```
